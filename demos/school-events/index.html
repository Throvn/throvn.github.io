Die MongoDB Console

... ist nur dazu da um zu überprüfen, ob ein Event welches vorher mit dem Add Event (oder Remove Event) Button erstellt wurde auch wirklich in der Datenbank gespeichert wurde. Bei jedem ersten einloggen mit in die Admin Console sollte dort so etwas wie: 8:22:22 PM: Database connected. stehen. In anderen Fällen wird das gesamte System nicht laufen, da die Verbindung zur Datenbank gestört ist. (Kontaktier mich dann am besten)

Die User Console

... ist nur für die Informatik-Gurus da. Lass dich von der nicht irritieren (nein wirklich jetzt). Auf die musst du nicht achten.

Die Anzeigen bedeuten folgendes:
  1. 6:22:22 PM: 4wd8...AAA requested data at position ["undefined location"] :
    Heißt nichts anderes, als das ein Gerät die Informationen aus der Datenbank beziehen möchte. Die undefined location ist noch zu sehen, da ich noch keine feste Geolocation einbauen wollte (sonst könnte man nur von bestimmten Koordinaten auf die Daten zugreifen).
  2. 6:22:22 PM: 4wd8...AAA recived all data :
    Bedeutet, dass das Gerät (das eben die Daten angefordert hat) die Daten bekommen hat.
  3. 6:43:24 PM: 4wd8...AAA joined the Admin Room :
    Meint, dass sich ein weiterer Nutzer in der Admin Console befindet. Dies kann ein zweiter Tab sein, der geöffnet ist, oder ein anderes Gerät. Ich würde immer empfehlen nur einen Tab gleichzeitig offen zu haben. Damit die Warnung überhaupt erst angezeigt wird, muss sich der "zweite Nutzer" auch mit dem richtigen Passwort angemeldet haben.
  4. 6:43:32 PM: 4wd8...AAA left because of transport close :
    Heißt, dass ein Nutzer die Verbindung zum Server abgebrochen/geschlossen hat. Sprich er hat die App geschlossen. Push-Benachrichtigungen werden allerdings immer noch angezeigt.

Connections (real-time)

Müsste relativ selbsterklärend sein. Zeigt wie viele Geräte gerade mit dem Server verbunden sind (inclusive der Admin Console).

Event List

Zeigt eine Liste aller Events an. Klickt man auf ein Event, so kann man im nachhinein bereits bestehende Events ändern. Alle Attribute sind veränderbar bis auf den Titel (wenn man diesen verändern will muss man ein neues Event erstellen und das alte löschen). Die Veränderung wird nicht live geupdated sondern tritt erst in Erscheinung, wenn die App neu geöffnet wird.

Add Event

Der Add Event Button öffnet ein Fenster, dass dir ein Formular bereitstellt. Alle Felder sind pflicht. Das Feld Icon zeigt hinterher ein Icon in der App an, um dem Benutzer das suchen nach beispielsweise Auftritten zu vereinfachen (music: )

Remove Event

Dieser Button löscht das vorher ausgewählte Event.

Send Notification

Der Knopf öffnet ein Fenster, mit dem man Push-Nachrichten verfassen und senden kann. Momentan kommen diese aber nur bei Android Geräten an. Für iOS braucht man (wie oben schon geschrieben) einen Developer-Account, der 100$ im Jahr kostet. Wenn man diesen hat, kann man allerdings die App auch im Appstore hochladen. Momentan müsste die App noch von der Website heruntergeladen werden.

Hilfe, alles explodiert!!!
